FAQ
Before you ask

The questions that came up enough to write down.

Is this safe? Will my account get banned? +
The tool uses the official VTU API — the same one your browser uses. It doesn't scrape, it doesn't spoof. Thousands of students have used it without incident. That said, use common sense: don't run it constantly and you should be fine.
Are my credentials stored anywhere? +
No. Your email and password are held in memory only for the duration of the job, then zeroed out. Nothing is written to disk or Redis. The source code is open — verify it yourself.
How long does it take? +
Typically 5–15 minutes per course depending on lecture count and queue depth. The server paces requests to avoid triggering rate limits, so patience pays off.
Can I close this tab? +
Yes. The job runs server-side. Copy the page URL (it has your job ID) and open it again to reconnect to a live job.
A lecture says "Max attempts reached" — what now? +
Some lectures don't accept progress via the API. Usually it's special or gated content. These can safely be ignored — VTU's portal typically doesn't require 100% to mark a course complete.
